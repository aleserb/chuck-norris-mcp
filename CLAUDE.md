# Chuck Norris MCP Server — Implementation Plan

## Overview
Build a vendor-neutral MCP (Model Context Protocol) server that serves random Chuck Norris facts. Works with Claude Code, Copilot, Gemini, ChatGPT, Cline, and any other MCP-compatible AI agent. Bonus: optional Claude Code plugin with `/chuck [category]` slash command.

## Architecture

### Core: MCP Server (npm package: `chuck-norris-mcp`)
**Vendor-neutral, works with any MCP-compatible client**

**Location:** `src/` directory  
**Launch method:** `node dist/index.js` (or `npx chuck-norris-mcp` after publishing)

- **Stdio transport:** Runs as a child process, communicates via JSON-RPC on stdin/stdout
- **Clients that can use it:** Claude Code, ChatGPT via Cline, GitHub Copilot, Gemini, and any tool supporting MCP
- **Two tools:**
  - `get_chuck_fact` — fetch a random fact (optional category filter, optional source: auto/local/api)
  - `list_categories` — list available categories

### Optional: Claude Code Plugin (convenience layer)
Adds a friendly `/chuck` slash command for Claude Code users specifically.

- `.claude-plugin/plugin.json` — Plugin manifest
- `.mcp.json` — Wires the MCP server for Claude Code
- `commands/chuck.md` — Slash command definition


## Implementation Steps

### 1. Create npm package scaffold
- `package.json` — name, deps, scripts, bin field
- `tsconfig.json` — ES2022, NodeNext, strict mode
- `tsup.config.ts` — bundle src/index.ts → dist/index.js with shebang

### 2. Build MCP server (`src/`)
**categories.ts**
- Export `CATEGORIES` array (dev, science, sport, movie, animal, food, history, money, celebrity)
- Export `Category` type
- Note: Only `dev` category has facts in initial version; others available for future expansion

**facts.ts**
- `LOCAL_FACTS` array with facts by category
  - **Dev category:** 82 facts from Jenkins chucknorris-plugin (https://github.com/jenkinsci/chucknorris-plugin)
  - **Other categories:** TBD (can be added later from api.chucknorris.io or other sources)
- `getRandomFact(category?)` function

**api.ts** (optional in Phase 1, extensible for Phase 2)
- `fetchFactFromApi(category?)` — calls https://api.chucknorris.io/jokes/random
- 3-second timeout, returns null on error
- Can be enabled later for non-dev categories

**server.ts**
- Create `McpServer` instance
- Register `list_categories` tool (no input schema)
- Register `get_chuck_fact` tool with input: category (optional enum), source (auto/local/api)
  - auto: tries API if category not dev, falls back to local DB (dev facts only available locally in Phase 1)
  - local: uses LOCAL_FACTS only (currently only dev category)
  - api: requires API success, throws if offline (extensible for future categories)

**index.ts**
- Shebang entry: `createServer()` + `StdioServerTransport`

## Key Design Decisions

**Why bundle MCP SDK & zod?** — `npx` downloads the package but does NOT run `npm install`. Bundling makes `dist/index.js` self-contained.

**Why Jenkins plugin facts for dev category?** — 82 high-quality programmer jokes, MIT-licensed, tested in production. Perfect offline-first local DB for Phase 1.

**Why phase the approach?** — Phase 1 delivers value immediately (dev facts only). Phase 2 can add other categories from api.chucknorris.io or other sources without disrupting Phase 1.

**Why stdio transport?** — Simplest MCP deployment: no HTTP server, no external process manager, Claude Code launches it on demand.

**Why local-first for dev?** — Dev category is always available offline. API integration for other categories is optional/future work.
